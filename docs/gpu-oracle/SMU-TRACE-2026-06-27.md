# Athena SMU trace — captured from the live amdgpu driver (2026-06-27)

Method: toggled Athena to amdgpu (`cmdline.athena-orig` + disable vfio modprobe +
`mkinitcpio -P` + reboot), armed a kprobe on `smu_cmn_send_smc_msg_with_param`
(`msg=%si:u32 param=%dx:u32`), captured the messages amdgpu sends, then restored
cold-vfio. The kprobe captures the **abstract** `enum smu_message_type` value + param.

## ASIC / SMU identification (decisive)
- GFX asic: **`gfx1101`** (Phoenix1, gfx 11.0.1).
- SMU: **`mp11304` = smu_v13_0_4** (Phoenix APU). So AthenaOS needs the
  **smu_v13_0_4** PPSMC message map — NOT smu1300.c (smu_v13_0_0 / Navi3x).
- SMU mailbox: MP1, `regMP1_SMN_C2PMSG_66` = msg, `C2PMSG_82` = param,
  `C2PMSG_90` = response.
- Working idle GFXCLK = **800 MHz always** (`pp_dpm_sclk: 0:800Mhz *`, even at
  `gpu_busy=0`). GFX is never unclocked when amdgpu is bound.

## Message NAMES present in this APU's firmware (from /sys/kernel/btf/amdgpu strings)
The KEY ones for GFX cold-start:
- **`SMU_MSG_PowerUpGfx`** ← likely the GFX-domain power-up AthenaOS is missing.
- `SMU_MSG_EnableGfxImu` (AthenaOS already sends this as PPSMC 0x16).
- `SMU_MSG_SetSoftMinGfxclk`, `SMU_MSG_SetSoftMaxGfxClk`, `SMU_MSG_SetHardMinGfxClk`.
- `SMU_MSG_GfxDriverResetRecovery`, `SMU_MSG_SetFclkGfxClkRatio`,
  `SMU_MSG_SetMinVideoGfxclkFreq`, `SMU_MSG_SetGfxclkOverdriveByFreqVid`.

## Runtime DPM clock messages (abstract enum → meaning by param)
Captured while forcing performance levels (param is the raw frequency in MHz):
| abstract msg | param range | meaning |
|---|---|---|
| 141 | 800 (min) | SetSoftMin **GFXCLK** |
| 140 | 2599 (max) | SetSoftMax **GFXCLK** |
| 143 / 142 | 500–1960 / 400–1200 | another clock (socclk/fclk) |
| 155 / 124 | 500–2057 / 400–1200 | another clock |
| 114 / 144 | 800/2057 AND 52428800/157286400 | mixed freq + size — capture noisy here |

GFXCLK is held in [800, 2599] by the DPM. The 800 minimum is what keeps the GFX
microengines clocked at idle.

## Post-reset GFX bring-up messages (gpu_recover via `cat .../amdgpu_gpu_recover`)
After the DPM noise, the actual re-init burst:
`msg=54 param=2`, `msg=16 param=128`, `msg=17 param=2019328`,
`msg=87/89/90/88 param=0` (repeating — feature enable/poll).

## GAPS (what's still needed to implement the fix)
1. **Numeric PPSMC values** for `PowerUpGfx` / `SetHardMinGfxClk` / `SetSoftMaxGfxClk`.
   The abstract enum (140/141/16/17/54/87-90) ≠ the numeric PPSMC id written to the
   mailbox. The lower send fns (`__smu_cmn_send_msg`) are inlined; bpftool/pahole are
   not on the Athena. Get them from upstream `smu_v13_0_4_ppsmc.h` (cross-check
   against a kprobe on the mailbox register WRITE: `amdgpu_device_wreg` filtered to
   the C2PMSG_66 offset).
2. **True cold-probe sequence** — the boot-time kprobe failed (`-22`: the amdgpu
   symbol isn't loaded when boot params are processed). Use a deferred module kprobe
   armed from an early systemd unit before amdgpu binds, or a `MODULE:SYMBOL` kprobe.

## The fix direction for AthenaOS (next implementation phase)
After first light, AthenaOS's SMU does only GetSmuVersion + DisallowGfxOff. amdgpu
additionally powers up + clocks the GFX domain. Add (numeric PPSMC values pending):
1. `PowerUpGfx` — power up the GFX block.
2. `SetHardMinGfxClk(800)` (+ `SetSoftMaxGfxClk(2599)`) — ramp/hold GFXCLK so the
   microengines are clocked.
Then re-test cold-vfio: the `CP_MES_GP3_LO` heartbeat should go non-zero (MES
executing), which unblocks the whole queue/CP-fetch chain already built.
